NU Store Management System API Reference

Flutter, Android, iOS বা অন্য যেকোন client app থেকে registration, login, dashboard, inventory master data, requisition submit, requisition list এবং requisition details ব্যবহারের জন্য সম্পূর্ণ API গাইড।

1. Overview

Base URL https://your-domain.test/api/v1
Response Format application/json
API Version v1
Primary Use Case Mobile app for NU store requisition workflow.

এই API দুই স্তরের authentication ব্যবহার করে: প্রথমে app-level token এবং এরপর login করা user-এর bearer token। Public master data endpoint-গুলো app token দিয়ে চলে; user-specific endpoint-এ app token এবং user token দুটোই লাগে।

ডকুমেন্টেশনটি যে domain/host থেকে খোলা হবে, Base URL স্বয়ংক্রিয়ভাবে সেই origin ব্যবহার করে /api/v1 path যোগ করবে। Local, staging এবং production environment-এ আলাদা করে URL edit করতে হবে না।

2. Quick Start for Flutter Developers

  1. Backend admin থেকে app token সংগ্রহ করুন।
  2. প্রতিটি request header-এ X-App-Token পাঠান।
  3. Login endpoint থেকে পাওয়া data.token Flutter secure storage-এ সংরক্ষণ করুন।
  4. Authenticated endpoints-এ Authorization: Bearer USER_TOKEN যোগ করুন।
  5. Initial app load-এ /settings, /departments, /designations, /categories, /products, /purposes cache করতে পারেন।
  6. Requisition submit করার আগে selected product, quantity এবং active purpose validate করুন।
const baseUrl = 'https://your-domain.test/api/v1';

final commonHeaders = {
  'Accept': 'application/json',
  'Content-Type': 'application/json',
  'X-App-Token': appToken,
};

final authHeaders = {
  ...commonHeaders,
  'Authorization': 'Bearer $userToken',
};

3. Authentication Model

TokenHeaderRequired ForNotes
App tokenX-App-Token: APP_TOKENAll API endpointsBackend setting হিসেবে configured token. Missing/invalid হলে 401।
User tokenAuthorization: Bearer USER_TOKEN/auth/me, /auth/profile, /auth/change-password, /auth/logout, /dashboard, /requisitions/auth/login response থেকে পাওয়া যায়। Logout করলে token delete হয়।
যদি app token Authorization header-এ পাঠানো হয়, authenticated endpoint-এ user token conflict হতে পারে। Flutter app-এ app token সবসময় X-App-Token header-এ পাঠানো recommended।

6.7 Requisition Visibility Rules

  • admin এবং super_admin সব requisition দেখতে পারে।
  • requisitioner শুধু নিজের requisition দেখতে পারে; mine=1 না দিলেও API তাকে own data-তেই সীমাবদ্ধ রাখে।
  • Departmental mode-এ store/approval roles নিজের department-এর requisitions দেখে।
  • Centralized mode-এ central store department-এর store roles সব department-এর requisitions দেখতে পারে।
  • Access rule fail করলে detail endpoint 404 return করতে পারে, যাতে hidden resource leak না হয়।

6.9 Recommended App-Side Settings Cache Rules

  • /settings response app startup-এ load করে memory/cache-এ রাখুন।
  • Logo URL null হলে local asset fallback ব্যবহার করুন।
  • Master data যেমন departments/designations/products/purposes cache করা যায়, কিন্তু requisition submit করার আগে stale product/purpose error handle করুন।
  • Admin settings change করলে user app refresh করলে নতুন settings পাবে; forced refresh button রাখতে পারেন।

12. Flutter Client Example

নিচের example একটি minimal API client দেখাচ্ছে। Production app-এ timeout, retry policy, logging এবং typed models যোগ করুন।

import 'dart:convert';
import 'package:http/http.dart' as http;

class NuStoreApiClient {
  NuStoreApiClient({required this.baseUrl, required this.appToken, this.userToken});

  final String baseUrl;
  final String appToken;
  final String? userToken;

  Map<String, String> get headers => {
        'Accept': 'application/json',
        'Content-Type': 'application/json',
        'X-App-Token': appToken,
        if (userToken != null) 'Authorization': 'Bearer $userToken',
      };

  Future<Map<String, dynamic>> login(String login, String password) async {
    final response = await http.post(
      Uri.parse('$baseUrl/auth/login'),
      headers: headers,
      body: jsonEncode({
        'login': login,
        'password': password,
        'device_name': 'flutter-app',
      }),
    );

    return _decode(response);
  }

  Future<Map<String, dynamic>> createRequisition(List<Map<String, dynamic>> items) async {
    final response = await http.post(
      Uri.parse('$baseUrl/requisitions'),
      headers: headers,
      body: jsonEncode({'items': items}),
    );

    return _decode(response);
  }

  Map<String, dynamic> _decode(http.Response response) {
    final body = jsonDecode(response.body) as Map<String, dynamic>;

    if (response.statusCode >= 400) {
      throw Exception(body['message'] ?? 'API request failed');
    }

    return body;
  }
}

14. Security & Production Notes

  • সবসময় HTTPS ব্যবহার করুন।
  • User token Flutter secure storage/keychain/keystore-এ রাখুন; plain shared preferences-এ রাখবেন না।
  • App token mobile binary থেকে extract করা সম্ভব, তাই প্রয়োজন হলে token rotate করুন এবং backend-side monitoring রাখুন।
  • 401 পেলে user token invalid হলে login screen-এ পাঠান; app token invalid হলে app configuration/support message দেখান।
  • 422 validation response থেকে field-wise error map করে form UI-তে দেখান।
  • Large lists-এর জন্য pagination metadata ব্যবহার করুন; সব requisition একসাথে load করবেন না।
Recommended release checklist: base URL configure, app token configure, login flow test, requisition submit test, offline cache strategy, token refresh/re-login UX, error state UI, empty state UI।
