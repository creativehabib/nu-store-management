NU Store Management System - Flutter API Documentation

এই ডকুমেন্টটি Flutter/Android/iOS অ্যাপ থেকে NU Store Management System ব্যবহার করার জন্য প্রয়োজনীয় API endpoint, header, request body এবং response structure দেখায়।

Base URL & Authentication

Base URL: https://nu-store-management.test/api/v1

সব request-এ server configured app token পাঠাতে হবে: X-App-Token: APP_TOKEN। Login-এর পর authenticated endpoint-এ user bearer token পাঠাতে হবে: Authorization: Bearer USER_TOKEN।
Accept: application/json
Content-Type: application/json
X-App-Token: APP_TOKEN
Authorization: Bearer USER_TOKEN

Authentication

POST /auth/register

নতুন requisitioner/user registration. Admin approval না হওয়া পর্যন্ত login করা যাবে না।

{
  "name": "Rahim Uddin",
  "email": "rahim@example.com",
  "pf_no": "PF-1001",
  "mobile_no": "01700000000",
  "department_id": 1,
  "designation_id": 2,
  "role": "requisitioner",
  "password": "Password#123",
  "password_confirmation": "Password#123"
}

POST /auth/login

{
  "login": "rahim@example.com or PF-1001",
  "password": "Password#123",
  "device_name": "android-phone"
}

Response থেকে data.token Flutter secure storage-এ রাখুন।

GET /auth/me

Logged-in user profile, department এবং designation সহ।

POST /auth/logout

বর্তমান device token revoke করে।

Dashboard

GET /dashboard?trend_filter=30

trend_filter হতে পারে 7, 15, 30, বা all। Role-based stats, chart labels/values, recent requisitions এবং user's own requisitions return করে।

Master Data

EndpointDescription
GET /inventoryDepartments, designations, categories, products, purposes, recent stock entries, recent requisitions একসাথে।

Requisitions

GET /requisitions

Authenticated user-এর role অনুযায়ী visible requisitions return করে। Query params: mine=1, status=pending, search=REQ, per_page=25।

POST /requisitions

Flutter app থেকে নতুন requisition submit করার endpoint।

{
  "items": [
    {"product_id": 1, "demanded_qty": 3, "purpose": "Official Use"},
    {"product_id": 2, "demanded_qty": 1, "purpose": "Training Purpose"}
  ]
}

Created requisition response-এ requisition_no, status, requester user এবং item/product details থাকে।

GET /requisitions/{id}

একটি requisition-এর full details, items, product category এবং approval history দেখায়।

Flutter Integration Example

final response = await http.post(
  Uri.parse('$baseUrl/auth/login'),
  headers: {
    'Accept': 'application/json',
    'Content-Type': 'application/json',
    'X-App-Token': appToken,
  },
  body: jsonEncode({
    'login': login,
    'password': password,
    'device_name': 'flutter-app',
  }),
);

final token = jsonDecode(response.body)['data']['token'];
final req = await http.post(
  Uri.parse('$baseUrl/requisitions'),
  headers: {
    'Accept': 'application/json',
    'Content-Type': 'application/json',
    'X-App-Token': appToken,
    'Authorization': 'Bearer $token',
  },
  body: jsonEncode({
    'items': [
      {'product_id': 1, 'demanded_qty': 2, 'purpose': 'Official Use'}
    ],
  }),
);

Security note: production app token publicly bundled in mobile apps can be extracted. Use HTTPS, rotate tokens periodically, and keep user bearer tokens in Flutter secure storage.
